写长篇小说，不再怕忘事

LucaWriter 是为长篇小说作者设计的写作工具。它帮你记住所有设定、人物和伏笔，让你专心讲故事，不用反复翻查前文。

艺术作品应该全部由人类完成。

这是 LucaWriter 的核心理念。AI 可以阅读、分析、给出建议，但不应对作品本身有任何写入操作。

基于这个原则，LucaWriter 的所有 AI 功能都围绕「读」来展开——阅读你的故事，记住每一个设定、人物与伏笔，在你有需要时给出反馈。

Why

「吃书」怎么办？

开始写小说后，几乎每个作者都会遇到这个问题：设定越来越多，人物关系越来越复杂，写到后面忘记前面写了什么。

没有工具时

每次提出新设定都要瞻前顾后。想不起某个伏笔的细节，需要手动翻找几十章。更新大纲会打断创作心流。写着写着剧情自相矛盾，回头修改费时费力。

有了 LucaWriter

只管大胆写。AI 会在适时主动提醒你可能的设定冲突。想不起什么设定时直接问 AI，立刻获得答案。大纲和记忆由 AI 自动维护，你不需要为管理它们而分心。

Features

核心功能

LucaWriter 不会替你写，但它会让你写得更稳。

摘要全书

逐章生成章节摘要，提取人物、事件与伏笔，自动合并为 source.md 全书摘要笔记和 outline.md 故事大纲。写完一章可触发增量摘要，或让 AI 自动识别。支持批量模式，多章合并一次调用，节省 API 次数。

记忆与大纲

AI 自动维护世界观、人物、时间线与关键事件，根据最新内容增量更新，防止长篇连载设定崩塌。AI 发现无法准确回答时会主动建议运行摘要。

深度分析

基于全书笔记模拟资深读者长评并预测剧情走向；自动梳理故事内时间线，支持子时间线展开。

即时反馈与标注

写作时 AI 实时分析并给出建议——不是替你写，而是帮你审视。支持在正文中添加多色荧光笔批注。支持推理模型的思维链展示，AI 的思考过程清晰可见。

书籍管理

书架视图管理多本书，支持改书名、导出、删除。每本书独立保存章节、大纲和对话记录。首次使用自动导入内置教程书本，快速上手。

多格式导入导出

导入支持 EPUB / TXT / MD / DOCX / PDF，自动识别章节标题拆分。导出支持 Markdown / TXT / ZIP / EPUB（可设置书名、作者、简介和封面）/ .lucawrite 加密完整备份。

本地模型一键下载

内置本地模型管理，支持一键下载 Gemma / Qwen 等模型，无需手动配置。搭配本地 Llama.cpp 预设，完全离线运行，数据不出电脑。

加密备份与安全

导出为 .lucawrite 格式时支持 AES-CBC 密码加密，完整保存章节、大纲、对话和标注。API Key 加密存储，访问范围可限制为本机或局域网。

Workflow

三步起步

部署只需一条命令，上手只需三步。

1

创建书本

在书架上新建空白书本，或直接导入现有的 EPUB / TXT / DOCX 等格式的小说文件，自动拆分章节。

2

安心写作

在简洁的纯文本编辑器中专注写作。需要时在右侧面板向 AI 搭档询问设定、人物或请求剧情建议。

3

交给 AI 摘要

写完一章点击「本章写完」，或积攒多章后运行「摘要全书」。AI 自动摘要、整理大纲、追踪伏笔。

容器

Docker 部署

适合有 Docker 环境的用户，数据持久化到宿主机。

docker build -t lucawriter . docker run -p 20000:20000 \ -v $(pwd)/usrdata:/app/usrdata \ lucawriter
数据保存在 usrdata/ 目录
